# Requirements: 英雄の旅 × “アイテム/舞台装置 bible” で映画品質へ（Immersive ride 優先）

## 背景 / 問題

- 現状は `assets.character_bible` と `assets.style_guide` が強い一方で、**竜宮城/玉手箱のような“物語の主役級アイテム・舞台装置”が、設定（bible）として構造化されていない**。
- その結果、scene prompt で都度の思いつきになりやすく、映画品質のディテール（材質・機構・ショー性・誘惑）が薄まりやすい。
- “映像内に文字は存在しない（字幕/看板/刻印で説明しない）”制約が、アイテム設計・scene設計まで一貫して降りていないため混入しやすい。

## ゴール

- `/toc-immersive-ride` の `video_manifest.md` を正本にして、キャラ同様に「object/setpiece」を設計 → 参照画像も先に生成 → sceneで参照できるようにする。
- 各アイテム/舞台装置について、
  - **映画の中での役割**（物語/感情/テーマに対する機能）
  - **映像から観客に与える情報**（言葉ではなく形/光/動きで伝える内容）
  - **メイン筋と無関係でも映像を魅力的にする要素**（ショー/仕掛け/誘惑のディテール）
  を設計として保持し、prompt に確実に反映できるようにする。
- 文字無し（No on-screen text）をシステムとして担保する（テンプレ/注入/禁止語彙）。

## 非ゴール / 除外

- 既存の出力 `output/` の生成物をコミットすること（`output/` は原則 gitignore）。
- 物語全体の脚本/構成の最適化（本タスクは “舞台装置/アイテム設計の契約” に集中）。
- 生成モデル選定・コスト最適化の再設計。

## 成功指標（MVP）

- `video_manifest.md` に `assets.object_bible` と `scenes[].image_generation.object_ids` が入り、生成時に自動注入される。
- object の参照画像が reference scene として必ず生成され、story scene の参照に使われる（破綻/揺れが減る）。
- “映画での役割 / 映像から与える情報 / ショー性”が、設計書（doc/playbook）として明文化され、作業者が迷わない。

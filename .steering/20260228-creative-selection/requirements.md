# Requirements: Creative Selection Policy (創造と選択)

## 背景 / 課題

現行の `docs/information-gathering.md` → `docs/story-creation.md` の流れでは、物語生成の手順が **Hero's Journey（英雄の旅）への当てはめ**を中心に書かれている。

しかし本プロジェクトの目的は「既に世の中で流行しているナラティブを映像化して高得点を取りに行く」ことであり、
物語価値の源泉は **登場人物** と **物語世界（世界観）** が生む多様性にある。

そのため、公式への当てはめ（理系的な検証）は“道具”に留め、**創造（多様性）→選択（スコア最適化）**を中核に置く必要がある。

また、複数文献に矛盾する記述がある場合、よりスコアが高い脚本のために **ハイブリッド（要素の混成）**を検討することがある。
これは破綻リスクが高いため、実行前に **人間承認が必須**とする。

## 目標

1) research→story→script の指示が、Hero's Journey を **絶対の公式として扱わない**こと  
2) 「創造（多様性を増やす）→選択（スコアが高い案を採る）」の判断軸を明示すること  
3) 矛盾する記述を **同一シーン内でハイブリッド化**する場合は、必ずユーザーに承認を求めること  
4) 承認が必要な場合の「質問テンプレ」と「安全チェック（矛盾/違和感）」を定義すること  

## 非目標（今回はやらない）

- コード（実装）側の自動ゲート追加（例: state に新キー追加して停止する等）
- 既存成果物（過去の `output/`）の一括更新
- 新しい評価システムの実装（スコア算出の自動化）

## 用語

- **創造（Creation）**: 登場人物/世界観/視点/解釈の候補を複数生成し、多様性を増やすこと
- **選択（Selection）**: もっともスコアが高い候補（または安全に統合できる候補）を採用すること
- **ハイブリッド（Hybridization）**: 矛盾する複数ソースの要素を混成し、単一のシーン/設定として提示すること

## 受け入れ条件（Acceptance Criteria）

- `docs/story-creation.md` に「創造と選択」の方針が明記され、Hero's Journey は **参考フレーム**として位置づく
- `docs/information-gathering.md` と `workflow/research-template*.yaml` で、矛盾点の整理と選択のための材料が出力できる
- `.claude/agents/deep-researcher.md` と `.claude/agents/director.md` が同方針を採用する
- ハイブリッドを行う場合に、ユーザーへ確認するための具体的な質問文（テンプレ）が docs/agents のどこかに存在する
